Discord Social SDK Go Wrapper Implementation Notes

Design Decisions

Modular Architecture: The wrapper is split into several files based on functionality (e.g., activity.go, lobby.go, voice.go). This improves maintainability and allows users to easily find relevant methods.

C Bridge (callbacks.c): A dedicated C file handles the mapping between C SDK callbacks and Go functions. This avoids cluttering the Go preamble and provides a clean separation between the Go wrapper and the underlying C API.

Callback Registry: A global, thread-safe registry (registry.go) manages Go callbacks, using unique IDs passed via the SDK's userData pointer. This is the standard way to bridge asynchronous C callbacks to Go.

Memory Management: Go's runtime.SetFinalizer is used extensively to call C _Drop functions, ensuring that opaque C structures are freed when the corresponding Go objects are garbage collected.

Deviations

None yet.

Trade-offs

String Handling: The C API uses Discord_String (pointer + size). I will convert these to Go string in getters and accept Go string in setters, performing the necessary C allocation/conversion internally. This makes the Go API idiomatic at the cost of some allocation overhead.

Open Questions

Linking: Should we bundle the DLLs/libs or expect the user to provide them? Current plan is to assume they are in the project structure as provided.

Callback Registry: How to best handle multiple Client instances and their callbacks? A global registry mapping IDs to Go objects is the standard approach.

Unimplemented Features

The following features from the C API (cdiscord.h) are currently NOT implemented in the Go wrapper:

  • Authorization System: Methods like Authorize, GetToken, RefreshToken, and OAuth2 flow handling.
  • Advanced Lobby Management:
    • Lobby metadata properties (Discord_Properties wrapping).
    • Member metadata.
    • Lobby events (LobbyDeleted, LobbyMemberAdded/Removed/Updated).
    • Channel linking/unlinking to lobbies.
  • Granular Voice Control:
    • Input/Output device selection and volume control.
    • Audio processing settings (AEC, AGC, Noise Suppression/Cancellation).
    • Raw audio capture and playback callbacks (UserAudioReceived, UserAudioCaptured).
  • Full Messaging Features:
    • Editing and deleting messages.
    • Lobby messages and message summaries.
    • Message event callbacks (MessageDeleted, MessageUpdated).
  • Social & Relationships:
    • Sending/Accepting friend requests.
    • Blocking/Unblocking users.
    • Searching friends by username.
    • Relationship event callbacks (RelationshipCreated/Deleted).
  • Activity Interaction:
    • Accepting and sending activity invites.
    • Join request handling.
    • Activity event callbacks (ActivityInvite, ActivityJoin).
  • System Integration:
    • Steam/Launch command registration.
    • Game window PID setting.
    • Check if Discord is installed.
